Accord

Contract-first integration layer for Web Component microfrontends

Accord enables type-safe mounting of remote custom elements in host applications. Define contracts using Zod schemas, generate TypeScript types automatically, and mount remote components with full runtime validation.

What is Accord?

Accord is a complete solution for building and integrating Web Component microfrontends. It provides three core packages that work together:

@accord/manifest

Define contracts using Zod schemas. Validate manifests and check version compatibility.

@accord/host

Runtime for loading and mounting remote components with prop validation and event handling.

@accord/types

CLI tool that generates TypeScript types from manifests for type-safe integration.

Key Concepts

  • ✓ Zod as single source of truth - Runtime validation, TypeScript types, and JSON schema all derive from Zod definitions
  • ✓ Host APIs via element property - Capabilities passed through element.host, not window globals
  • ✓ Semver contract negotiation - Version compatibility via semver.satisfies
  • ✓ Client-side only - No SSR for remotes, simplifying architecture

Why Accord?

Traditional Web Component integration has several pain points that Accord addresses:

Type Safety Gap

Remote components loaded at runtime lack TypeScript types on the host. Accord generates types automatically from manifests.

Contract Ambiguity

No clear specification of what props, events, and capabilities a component needs. Accord defines explicit contracts using Zod schemas.

Runtime Validation

Props and events are passed without validation, leading to runtime errors. Accord validates in dev mode using the manifest schemas.

Global Pollution

Host services typically injected via window.*, creating tight coupling. Accord passes capabilities through the element's host property.

Version Negotiation

No standard way to handle contract version compatibility. Accord uses semver-based version matching.

Getting Started

Installation

# For remote component development
npm install @accord/manifest zod

# For host application
npm install @accord/host @accord/types

Step 1: Define a Manifest (Remote)

Create a manifest file in your remote component that defines the contract:

// csx-user-card.manifest.ts
import { z } from "zod";
import { defineManifest } from "@accord/manifest";

export const manifest = defineManifest({
  meta: {
    name: "csx-user-card",
    version: "1.0.0",
    contractVersion: "1.0.0",
    tagName: "csx-user-card"
  },
  props: z.object({
    userId: z.string(),
    readonly: z.boolean().default(false)
  }),
  events: {
    "user:selected": z.object({ userId: z.string() }),
    "user:deleted": z.object({
      userId: z.string(),
      reason: z.enum(["manual", "policy"])
    })
  },
  capabilities: {
    audit: z.object({
      log: z.function()
        .args(z.object({
          action: z.string(),
          entity: z.string(),
          metadata: z.record(z.any()).optional()
        }))
        .returns(z.void())
    }),
    nav: z.object({
      navigate: z.function()
        .args(z.object({ to: z.string() }))
        .returns(z.void())
    })
  },
  hostRequirements: {
    styling: "no-shadow-dom",
    designTokens: true
  }
});

Step 2: Build Your Remote Component

Implement your Web Component following the manifest contract:

// csx-user-card.ts
import { LitElement, html } from "lit";
import { customElement, property } from "lit/decorators.js";

@customElement("csx-user-card")
export class CsxUserCard extends LitElement {
  @property({ type: String }) userId = "";
  @property({ type: Boolean }) readonly = false;

  // Disable shadow DOM per hostRequirements
  protected createRenderRoot() {
    return this;
  }

  private handleSelect() {
    this.dispatchEvent(
      new CustomEvent("user:selected", {
        detail: { userId: this.userId },
        bubbles: true,
        composed: true
      })
    );
  }

  render() {
    return html`
      <div>
        <strong>User:</strong> ${this.userId}
        <button ?disabled=${this.readonly} @click=${this.handleSelect}>
          Select
        </button>
      </div>
    `;
  }
}

Step 3: Generate Types (Host)

Use the CLI to generate TypeScript types from the manifest:

# From local manifest file
npx accord-types --manifest ./src/csx-user-card.manifest.ts --outDir ./src/generated

# From remote manifest URL
npx accord-types --manifestUrl https://cdn.example.com/manifest.json --outDir ./src/generated

Step 4: Mount the Component (Host)

In React 19+, you can use custom elements directly in JSX:

// layout.tsx - Load the remote script
import Script from "next/script";

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <head>
        <Script
          src="https://cdn.example.com/csx-user-card.js"
          strategy="beforeInteractive"
        />
      </head>
      <body>{children}</body>
    </html>
  );
}

// page.tsx - Use the component
export default function HomePage() {
  return (
    <main>
      <csx-user-card userId="user-123" readonly={false} />
    </main>
  );
}

Usage

Manifest Definition

A manifest defines the complete contract between a remote component and its host:

Section Description
meta Component name, version, contract version, and HTML tag name
props Zod schema for component input properties
events Record of event names to Zod schemas for payloads
capabilities Host APIs the component requires (optional)
hostRequirements Styling mode and design token requirements (optional)

Imperative Mounting

For non-React hosts or dynamic mounting scenarios:

import { registerRemote, mount } from "@accord/host";

// Register the remote component
registerRemote({
  id: "csx-user-card",
  url: "https://cdn.example.com/csx-user-card.js",
  integrity: "sha384-..." // Optional SRI hash
});

// Mount the component
const { element, unmount } = await mount({
  remoteId: "csx-user-card",
  tagName: "csx-user-card",
  container: document.getElementById("slot"),

  props: {
    userId: "user-123",
    readonly: false
  },

  // Pass host capabilities
  hostApi: {
    audit: {
      log: (event) => console.log("audit", event)
    },
    nav: {
      navigate: (input) => window.location.href = input.to
    }
  },

  // Handle events
  onEvent: (eventName, payload) => {
    console.log("event", eventName, payload);
  },

  // Fallback on failure
  fallback: "Component failed to load",

  // Enable dev-time validation
  dev: true,
  manifest: { props: propsSchema, events: eventsSchema },

  // Timeout
  timeoutMs: 10000
});

// Cleanup when done
unmount();

Host API Capabilities

The host can provide these standard capabilities to remote components:

type HostApi = {
  auth?: {
    getToken: () => Promise<string>;
  };
  audit?: {
    log: (event: { action: string; entity: string; metadata?: Record<string, unknown> }) => void;
  };
  nav?: {
    navigate: (input: { to: string }) => void;
  };
  i18n?: {
    t: (key: string, params?: Record<string, string>) => string;
  };
  theme?: {
    tokens: () => Record<string, string>;
  };
  store?: {
    subscribe: (listener: (value: unknown) => void) => () => void;
    getSnapshot: () => unknown;
  };
};

Type Generation CLI

The accord-types CLI supports two workflows:

# From local TypeScript manifest
accord-types --manifest ./src/component.manifest.ts --outDir ./src/generated

# From remote JSON manifest
accord-types --manifestUrl https://cdn.example.com/manifest.json --outDir ./src/generated

# With custom output filename
accord-types --manifest ./src/component.manifest.ts --outDir ./src/generated --fileName types.ts

Complete Remote Component (Lit)

import { z } from "zod";
import { defineManifest } from "@accord/manifest";

export const manifest = defineManifest({
  meta: {
    name: "csx-user-card",
    version: "1.0.0",
    contractVersion: "1.0.0",
    tagName: "csx-user-card"
  },
  props: z.object({
    userId: z.string(),
    readonly: z.boolean().default(false)
  }),
  events: {
    "user:selected": z.object({ userId: z.string() }),
    "user:deleted": z.object({
      userId: z.string(),
      reason: z.enum(["manual", "policy"])
    })
  },
  capabilities: {
    audit: z.object({
      log: z.function()
        .args(z.object({
          action: z.string(),
          entity: z.string(),
          metadata: z.record(z.any()).optional()
        }))
        .returns(z.void())
    })
  },
  hostRequirements: {
    styling: "no-shadow-dom",
    designTokens: true
  }
});

// csx-user-card.ts
import { LitElement, html, css } from "lit";
import { customElement, property } from "lit/decorators.js";

@customElement("csx-user-card")
export class CsxUserCard extends LitElement {
  static styles = css`
    :host {
      display: block;
      font-family: sans-serif;
      border: 1px solid #e2e8f0;
      padding: 16px;
      border-radius: 8px;
    }
    button { margin-top: 12px; }
  `;

  @property({ type: String }) userId = "";
  @property({ type: Boolean }) readonly = false;

  protected createRenderRoot() {
    return this; // No shadow DOM
  }

  private handleSelect() {
    // Access host capabilities
    (this as any).host?.audit?.log({
      action: "select",
      entity: "user",
      metadata: { userId: this.userId }
    });

    this.dispatchEvent(
      new CustomEvent("user:selected", {
        detail: { userId: this.userId },
        bubbles: true,
        composed: true
      })
    );
  }

  private handleDelete() {
    this.dispatchEvent(
      new CustomEvent("user:deleted", {
        detail: { userId: this.userId, reason: "manual" },
        bubbles: true,
        composed: true
      })
    );
  }

  render() {
    return html`
      <div>
        <strong>User:</strong> ${this.userId}
        <div>
          <button ?disabled=${this.readonly} @click=${this.handleSelect}>
            Select
          </button>
          <button ?disabled=${this.readonly} @click=${this.handleDelete}>
            Delete
          </button>
        </div>
      </div>
    `;
  }
}

Validate Manifest at Build Time

import { validateManifest } from "@accord/manifest";
import { manifest } from "./csx-user-card.manifest";

const result = validateManifest(manifest);

if (!result.success) {
  console.error("Manifest validation failed:", result.errors);
  process.exit(1);
}

console.log("Manifest is valid!");

Check Version Compatibility

import { resolveCompatibility } from "@accord/manifest";

// Host requires contract version ^1.0.0
const hostContractRange = "^1.0.0";

// Component provides contract version 1.2.0
const componentContractVersion = "1.2.0";

const isCompatible = resolveCompatibility(
  hostContractRange,
  componentContractVersion
);

console.log("Compatible:", isCompatible); // true

API Reference

@accord/manifest

Export Type Description
defineManifest(manifest) Function Type-safe manifest definition helper (identity function)
validateManifest(manifest) Function Validates manifest structure, returns { success, errors }
resolveCompatibility(hostRange, componentVersion) Function Checks semver compatibility between host and component
getManifestJsonSchema(manifest) Function Converts manifest to JSON Schema for remote hosting
contractMetaSchema Zod Schema Schema for validating the meta section
hostRequirementsSchema Zod Schema Schema for validating host requirements

@accord/host

Export Type Description
registerRemote(registration) Function Register a remote component by ID and URL
loadRemote(id, timeoutMs?) Async Function Load remote script with deduplication and caching
mount(options) Async Function Mount component, returns { element, unmount }
RemoteRegistration Type { id, url, integrity?, versionRange? }
MountOptions Type Options for the mount() function
MountHandle Type { element, unmount } returned by mount()
HostApi Type Standard host capabilities interface

@accord/types CLI

Option Description
--manifest <path> Path to local TypeScript manifest file
--manifestUrl <url> URL to remote manifest.json
--outDir <path> Output directory for generated types (required)
--fileName <name> Custom output filename (optional)
